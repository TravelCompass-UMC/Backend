# Travel Compass Backend


## 브랜치 전략
* **Git-flow** 베이스의 브랜치 전략을 사용합니다.
* main, feature, develop 브랜치만 유지합니다. (필요시 hotfix 브랜치 사용)
* feature의 경우 `feature/1`과 같이 이슈 번호를 함께 작성합니다.
* PR을 거쳐 병합된 feature 브랜치는 삭제합니다. (merge를 수행한 사람이 삭제해주세요!)


## GitHub Projects
* 모든 기능 구현 및 수정에는 이슈를 먼저 만든 후 수행합니다.
* 칸반보드에서 각 이슈들을 진행상황에 맞게 구분하여 관리합니다.
  * Ready
  * In Progress
  * In Review
  * Done
> 꼭 이슈와 칸반보드를 연동합시다!

## Issue 컨벤션
* **Projects**에서 Issue를 생성합니다. (Projects의 Assigness에도 본인을 태그해주세요.)
* Assigness에 본인을 태그합니다.
* 적절한 Labels을 붙입니다.
* Development에 작업한 브랜치를 연결합니다. (PR의 머지가 성공적으로 수행된 경우 해당 Issue를 자동으로 close 해줍니다.)

### 제목
* Commit의 제목 컨벤션과 동일합니다.

### 본문
* 지정된 Issue 템플릿에 맞추어 작성합니다. (실제 작성시 `e.g.` 부분은 삭제합니다.)


## Commit 컨벤션
* 아래의 예시와 같이 제목, 본문, 꼬리말의 3개의 파트로 나누어 Commit 메시지를 작성합니다.
* 설명이 충분하다면 제목만으로도 괜찮습니다.
* 각 파트는 개행(`\n`)으로 구분해야 합니다.
```
Feat: "로그인 기능 구현"

새로고침 시 로그인 유지 기능 구현

Resolves: #30
Ref: #20
```

### 제목
* 위 예시와 같이 type과 제목을 콜론+빈칸(`: `)으로 분리합니다.
* 제목은 반각(영어)기준 50자, 전각(한글)기준 25자 이내로 작성합니다.
* type의 첫 글자는 대문자로 작성합니다.
* 제목 끝에 마침표(`.`)를 붙이지 않습니다.
* 명령식, 개조식으로 작성합니다.

#### 제목의 type 종류
| type | 설명 |
|:------|:------|
|Feat|새로운 기능 구현|
|Fix|버그 수정|
|Refactor|프로덕션 코드 리펙토링|
|Docs|문서 수정|
|Test|테스트 추가, 테스트 리펙토링|
|Chore|패키지 매니저(gradle) 설정 및 수정, 빌드 테스트 업데이트|
|CI|CI/CD 관련 작업|
|Rename|파일 or 폴더의 이름 혹은 위치 수정|
|Remove|파일 or 폴더 삭제|
|Style|코드 포멧 변경 및 간격 변경 등의 자잘한 수정, 코드 수정이 없는 경우|
> 진짜 자잘한 수정이라 커밋 제목을 작성하기 귀찮은 경우 `Style: WIP`와 같이 작성합니다.
> (WIP : Work In Process)

### 본문
* 부연설명 혹은 커밋의 이유를 작성합니다.
* 반각기준 72자를 넘기지 않습니다.
* 어떻게 변경했는지 보다 **무엇을 변경했는지, 왜 변경했는지**를 작성합니다.

### 꼬리말
* GitHub 이슈의 tracker id를 작성합니다.
* 제목과 같이 type과 내용을 콜론+빈칸(`: `)으로 분리합니다.

#### 꼬리말의 type 종류
| type | 설명 |
|:------|:------|
|Fixes|수정 중인 이슈|
|Resolves|해결된 이슈|
|Ref|참고할 이슈|


## Pull Request 컨벤션
* Reviewers에는 리뷰 요청할 사람을, Assignees에는 본인을 태그합니다.
* 적절한 Labels을 붙입니다.
* Development에 작업한 브랜치를 연결합니다. (머지가 성공적으로 수행된 경우 해당 PR을 자동으로 close 해줍니다.)

### 제목
* 명령식, 개조식으로 작성합니다.
* 아직 작업중인 PR인 경우 `[WIP]`를 제목 앞에 붙이고, 작업이 끝났다면 이를 제거합니다.
```
[WIP] 로그인 기능 구현
```

### 본문
* 지정된 PR 템플릿에 맞추어 작성합니다. (실제 작성시 `e.g.` 부분은 삭제합니다.)
